One helpful thing I've started to during #adventofcode is implementing my solution as unit tests. That way I can first make the example pass (with a known answer), and then ensure it keeps passing as I do any modifications necessary for the full input. When I get the puzzle result confirmed, I add an expectation for the full input test as well. Then I repeat that for the second puzzle of the day.
