@satnam6502 Completely gassed that this is actual well typed code (for Batcher's even-odd mergesort):

merge<n> : ((T, T) -> (T, T)) -> S F n of T -> S F n of T -> S F S n of T
merge casT [a] [b] = let (l, h) = casT (a, b) in [l, h]
merge casT (le =%= lo) (re =%= ro) =
 let (min ,- es, os -, max) = (merge casT le re, merge casT lo ro) in
  let (es2, os2) = unzip (map casT (zip es os)) in
   min ,- (es2 =%= os2) -, max

sort<n> : ((T, T) -> (T, T)) -> S F n of T -> S F n of T
sort _ [a] = [a]
sort casT (xs =&= ys) = merge casT (sort casT xs) (sort casT ys)

(S is successor; F is "full" (predecessor-of-two-to-the), =%= deals/riffles alternating elements of vectors, =&= chops/glues halves of vectors.)

What's joyful about it is that the typechecker which does the numbers is outrageously cheap.
